## 2. 계정 영속 실기기 검증 (필수 — docs/account-persistence-design.md 구현 순서 3)

- [ ] **iOS**: 설치 → 스터디룸 방 입장 → 앱 삭제 → 재설치 → 같은 uid(닉네임/자리 유지) 확인

## 6.5 ★다음 빌드 릴리스 노트 수정 (필수) — 과대약속 제거

**문제**: 1.0.36 릴리스 노트("이 버전에서 업그레이드된 사항")에 아래 블록이 실려 나감(이미 배포됨,
소급 수정 불가). 안드로이드 자동백업은 조건부(24h·Wi-Fi·충전·구글계정)라 "재설치해도 유지"를
**단정하면 안 됨** — 작동 안 하면 사용자가 데이터 잃고 배신감만. (설명(Description) 필드엔 없음, 릴리스 노트 한정)

```
■ 재설치해도 안전하게
앱을 지웠다 다시 설치해도 공부 기록과 설정이 유지되도록 개선했어요.
```

**조치**: iOS·Android **둘 다 1.0.37 릴리스 노트에서 삭제**(2026-07-23 결정). 소급 수정은
불가 확인됨 — iOS 1.0.36은 배포 준비됨 상태라 잠김, Android vc64(1.0.37)도 심사 제출본이라 편집 불가.
따라서 **다음에 노트를 새로 쓰는 시점(iOS 1.0.37 빌드 제출 / Android 그 다음 릴리스)** 에서 아래 블록을 뺀다.
남기고 싶으면 자동유지 약속 대신 수동 백업 유도로:
```
■ 백업이 더 안심돼요
설정 탭에서 공부 기록을 파일로 백업·복원할 수 있어요. 기기를 바꾸거나 앱을 다시 설치하기 전에 백업해 두세요.
```
- 참고: 웹사이트 FAQ(index.html)는 이미 정직함("기기 교체·재설치 전 백업 권장") → 이 톤으로 통일.
- allowBackup:true 자체는 유지(조용한 best-effort 보너스, 콜드스타트 복원 방어 있어 안전) — 문구만 문제.
