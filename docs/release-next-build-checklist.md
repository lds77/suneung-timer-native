## 2. 계정 영속 실기기 검증 (필수 — docs/account-persistence-design.md 구현 순서 3)

- [ ] **iOS**: 설치 → 스터디룸 방 입장 → 앱 삭제 → 재설치 → 같은 uid(닉네임/자리 유지) 확인
- [ ] iOS: 콘솔에서 SecureStore 2048바이트 초과 경고가 뜨는지 + 그래도 저장되는지
- [ ] **Android**: 설치 → 방 입장 → `adb shell bmgr backupnow com.yeolgong.timer`로 백업 강제
      → 삭제 → 재설치 → uid+공부 데이터 복원 확인 (자동백업은 평소 24h 주기·Wi-Fi 조건)
- [ ] 기존 사용자 마이그레이션: 구버전에서 업그레이드한 기기에서 첫 실행 후 삭제→재설치해도 uid 유지
      (읽기 경로의 키체인 미러가 작동했는지)
