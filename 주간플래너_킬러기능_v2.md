# 열공메이트 — 주간 플래너 (킬러 기능) 구현 프롬프트

## ⚠️ 작업 전 필수

프로젝트의 현재 파일들을 먼저 **전부** 읽고 분석해. 디자인과 기능이 이전과 달라졌을 수 있어.
특히 아래 파일들의 **현재 상태**를 완전히 파악한 후 작업 시작:

```
src/hooks/useAppState.js    — 전역 상태, 타이머 로직 (가장 중요)
src/screens/FocusScreen.js  — 집중탭 UI
src/screens/SettingsScreen.js — 설정탭
src/screens/SubjectsScreen.js — 과목탭 (학습법, 루틴)
src/screens/StatsScreen.js  — 통계탭
src/utils/storage.js        — AsyncStorage 래퍼
src/constants/colors.js     — getTheme() 테마 시스템
src/constants/presets.js    — 과목 프리셋, 집중밀도 등급
src/constants/characters.js — 캐릭터 시스템
App.js                      — 네비게이션 구조
```

기존 코드 스타일, getTheme() 시스템, 컴포넌트 패턴을 **그대로** 따라서 작업해.

## 왜 이 기능이 킬러인가

경쟁 앱(열품타, 포레스트, 플립)은 전부 **"타이머만"** 제공함.
"오늘 뭘 공부할지"는 사용자가 알아서 정해야 하고, "언제 공부할 수 있는지"도 사용자가 머릿속으로 계산해야 함.

우리 앱은 **"시간표를 보고 → 빈 시간을 알고 → 계획을 세우고 → 원탭으로 실행하고 → 달성률을 확인하는"** 전체 흐름을 하나의 앱 안에서 해결함.

이건 다른 앱에 없는 기능이고, 타이머 앱과 플래너 앱을 따로 쓸 필요가 없어지는 기능임.

## 핵심 설계 원칙 (모든 코드에 이 원칙 적용)

### 원칙 1: 사용자는 자기 카테고리만 본다
6개 카테고리가 있지만, 사용자는 처음에 한 번 선택하면 그 이후로 자기에게 맞는 화면만 봄.
초등학생이 공시생 루틴을 볼 일 없고, 대학생이 중학생 시간표를 볼 일 없음.
**복잡도는 코드 안에만 존재하고, 화면은 항상 단순해야 함.**

### 원칙 2: 30초 안에 설정 완료
처음 플래너를 켤 때 학교급에 맞는 기본 템플릿이 자동 세팅되고,
사용자는 "내 상황에 안 맞는 것만 수정"하면 됨.
빈 화면에서 처음부터 입력하게 하면 이탈함.

### 원칙 3: 강제하지 않는다
플래너를 안 쓰는 사용자는 기존과 100% 동일한 경험.
플래너를 쓰더라도 "계획 안 지켰다"고 부정적 피드백 절대 하지 않음.
"오늘도 조금이라도 했으니 대단해!" 톤 유지.

### 원칙 4: 계획 → 실행이 원탭
계획에서 타이머 시작까지 **탭 1번**. 과목, 시간, 색상이 자동 세팅.
이게 안 되면 그냥 즐겨찾기 쓰는 것과 차이가 없음.

## 1. 사용자 카테고리 확장

### 기존 schoolLevel 확장

```javascript
// settings.schoolLevel 값 변경
// 기존: 'elementary' | 'middle' | 'high'
// 변경: 'elementary_lower' | 'elementary_upper' | 'middle' | 'high' | 'nsuneung' | 'university' | 'exam_prep'
```

| 코드값 | 표시명 | 대상 |
|--------|--------|------|
| elementary_lower | 초등 저학년 | 1~3학년 |
| elementary_upper | 초등 고학년 | 4~6학년 |
| middle | 중학생 | 중1~3 |
| high | 고등학생 | 고1~3 |
| nsuneung | N수생 | 재수/삼수/반수 |
| university | 대학생 | 대학교 재학생 |
| exam_prep | 공시생/자격증 | 공무원, 자격증, 편입 등 |

**기존 'elementary'는 'elementary_upper'로 마이그레이션.**
기존 사용자가 elementary였으면 자동으로 elementary_upper로 변환.

### 설정 화면 UI

현재 학교급 선택 UI를 확장. 2줄 배치 권장:
```

[N수생]      [대학생]     [공시생/자격증]
```

또는 세로 리스트에 아이콘:


### 자동 저장 (기존 useEffect에 weeklySchedule 추가)

### 헬퍼 함수들

```javascript
// 오늘 요일 키 반환
const getDayKey = () => {
  return ['sun', 'mon', 'tue', 'wed', 'thu', 'fri', 'sat'][new Date().getDay()];
};

// 오늘의 스케줄 반환
const getTodaySchedule = useCallback(() => {
  if (!weeklySchedule || !weeklySchedule.enabled) return null;
  const dayData = weeklySchedule[getDayKey()];
  return dayData || { fixed: [], plans: [] };
}, [weeklySchedule]);

// 특정 plan의 오늘 완료 시간 (초) 반환
const getPlanCompletedSec = useCallback((planId) => {
  const today = getToday();
  return sessions
    .filter(s => s.date === today && s.planId === planId)
    .reduce((sum, s) => sum + (s.durationSec || 0), 0);
}, [sessions]);

// 오늘 전체 달성률 (%) 반환
const getTodayPlanRate = useCallback(() => {
  const today = getTodaySchedule();
  if (!today || !today.plans || today.plans.length === 0) return null;
  const totalTarget = today.plans.reduce((sum, p) => sum + p.targetMin * 60, 0);
  if (totalTarget === 0) return null;
  const totalDone = today.plans.reduce((sum, p) => sum + getPlanCompletedSec(p.id), 0);
  return Math.min(100, Math.round(totalDone / totalTarget * 100));
}, [getTodaySchedule, getPlanCompletedSec]);

// 계획에서 타이머 시작
const startFromPlan = useCallback((plan) => {
  addTimer({
    type: 'countdown',
    label: `${plan.icon || ''} ${plan.label}`.trim(),
    color: plan.color,
    subjectId: plan.subjectId || null,
    totalSec: plan.targetMin * 60,
    planId: plan.id,  // 세션 기록에 태깅
  });
}, [addTimer]);

// 가용 시간 계산 (분 단위)
const getAvailableMin = useCallback((dayKey) => {
  if (!weeklySchedule) return 24 * 60;
  const dayData = weeklySchedule[dayKey];
  if (!dayData || !dayData.fixed || dayData.fixed.length === 0) return 24 * 60;
  
  const fixedMin = dayData.fixed.reduce((sum, f) => {
    const [sh, sm] = f.start.split(':').map(Number);
    const [eh, em] = f.end.split(':').map(Number);
    return sum + (eh * 60 + em) - (sh * 60 + sm);
  }, 0);
  return Math.max(0, 24 * 60 - fixedMin);
}, [weeklySchedule]);
```

### addTimer 수정 — planId 필드 추가

타이머 객체 생성 부분에 `planId: opts.planId || null` 추가.

### recordSessionInternal 수정 — planId 저장

세션 객체에 `planId: opts.planId || null` 추가.
fireComplete에서 recordSessionInternal 호출 시 `planId: t.planId` 전달.


```
[월] [화] [수] [목] [금] [토] [일]
```
- 가로 스크롤 탭, 선택된 요일 강조 (accentColor)
- 오늘 요일에 작은 점 표시

#### 각 요일의 내용

**A. 고정 일정 섹션**
```
🏫 고정 일정
┌─────────────────────────────────┐
│ 🏫 학교        08:30 ~ 15:30  ✕│
│ 🏢 수학학원    18:00 ~ 20:00  ✕│
└─────────────────────────────────┘
[+ 고정 일정 추가]
```

[+ 고정 일정 추가] 누르면 모달:
1. 유형 선택 (FIXED_TYPES에서 가져옴, 가로 스크롤 칩)
2. 이름 (유형 선택하면 자동 입력, 수정 가능)
3. 시작 시간 (30분 단위 피커 또는 시간 선택 드롭다운, 06:00~24:00)
4. 끝 시간 (시작 시간 이후만 선택 가능)
5. [추가] 버튼

**B. 가용 시간 표시 (정보만, 입력 아님)**
```
🟢 공부 가능 시간: 6시간 30분
   06:00~08:30 (2h30m) | 15:30~18:00 (2h30m) | 20:00~22:00 (2h)
```
고정 일정이 바뀔 때마다 자동 재계산.

**C. 공부 계획 섹션**
```
📝 공부 계획                    합계: 2시간 10분
┌─────────────────────────────────┐
│ 📐 수학    40분    [▲][▼]  ✕  │
│ 📗 영어    30분    [▲][▼]  ✕  │
│ 📘 국어    30분    [▲][▼]  ✕  │
│ 📝 복습    30분    [▲][▼]  ✕  │
└─────────────────────────────────┘
[+ 과목 추가]
```

- [▲][▼] 순서 변경 버튼
- [✕] 삭제 버튼
- 합계 시간이 가용 시간 초과 시:
  `⚠️ 계획 3h30m > 가용 2h30m — 조금 줄여볼까요?`
  (경고만, 등록 자체는 막지 않음)

[+ 과목 추가] 누르면 모달:
1. 과목 선택:
   - **내 과목에서 선택** (subjects 배열, 이미 등록된 과목 목록)
   - **직접 입력** (이름, 아이콘 선택, 색상 선택)
2. 목표 시간: Stepper로 5분 단위 (10분~180분, 기본 30분)
3. [추가] 버튼

#### 하단: 요일 복사

```
[📋 이 요일을 다른 요일에 복사]
```
누르면: 체크박스로 복사할 요일 선택 (월~일 중 현재 요일 제외)
확인하면: fixed + plans 모두 복사 (각 항목의 id는 새로 생성)

## 6. 오늘의 계획 카드 (FocusScreen.js)

### 위치
캐릭터 영역 아래, 즐겨찾기 영역 위.
(기존 UI 요소 사이에 자연스럽게 삽입)

### 표시 조건
`weeklySchedule?.enabled === true` 이고 오늘 plans가 1개 이상일 때만 표시.
조건 안 맞으면 카드 자체가 렌더링 안 됨 (기존 화면 그대로).

### 카드 디자인

```
┌─────────────────────────────────────┐
│ 📅 월요일 오늘의 계획   2/4   [편집]│
│                                     │
│ ┃ 🏫 학교 08:30~15:30              │  ← 고정 일정 (세로선+회색, 탭 불가)
│ ┃ 🏢 수학학원 18:00~20:00          │
│ ──────────────────────────────────  │
│ ✅ 📐 수학 40분     42분/40분      │  ← 완료 (취소선 + 실제/목표)
│ 🔵 📗 영어 30분     12분/30분  [▶] │  ← 진행 중 (미니 진행률 바)
│ ⬜ 📘 국어 30분                [▶] │  ← 미시작
│ ⬜ 📝 복습 30분                [▶] │
│                                     │
│ ████████░░░░░░░░  42%  54분/2h10m  │  ← 전체 진행률 바
└─────────────────────────────────────┘
```

### 상태별 표시

| 상태 | 조건 | 표시 |
|------|------|------|
| 완료 ✅ | completedSec >= targetMin * 60 * 0.8 | 취소선 + "42분/40분" + 체크 |
| 진행 중 🔵 | 0 < completedSec < 80% | 미니 진행률 바 + "12분/30분" + [▶] |
| 미시작 ⬜ | completedSec === 0 | [▶] 버튼만 |

### 동작

- **[▶] 버튼:** `app.startFromPlan(plan)` 호출 → 카운트다운 타이머 즉시 생성
- **[편집] 버튼:** ScheduleEditorScreen으로 이동 (오늘 요일 선택 상태로)
- **카드 접기/펼치기:** 제목 영역 탭하면 토글
  - 접힌 상태: "📅 월요일 오늘의 계획 2/4 ████░░ 42%"  (한 줄)
  - 펼친 상태: 전체 표시 (기본값)

### 타임라인 배치 규칙

**명확한 규칙: 고정 일정을 시간순으로 상단에 모아서 표시하고, 구분선 아래에 공부 계획을 order순으로 나열한다.**

```javascript
const buildTimeline = (fixed, plans) => {
  const timeline = [];
  
  // 1. 고정 일정 시간순 (맥락 정보)
  const sortedFixed = [...fixed].sort((a, b) => a.start.localeCompare(b.start));
  sortedFixed.forEach(f => timeline.push({ type: 'fixed', data: f }));
  
  // 2. 구분선 (고정 일정이 있을 때만)
  if (sortedFixed.length > 0 && plans.length > 0) {
    timeline.push({ type: 'divider' });
  }
  
  // 3. 공부 계획 order순 (체크리스트)
  const sortedPlans = [...plans].sort((a, b) => a.order - b.order);
  sortedPlans.forEach(p => timeline.push({ type: 'plan', data: p }));
  
  return timeline;
};
```

## 7. 스마트 알림 (플래너 연동)

### 동작 원리
고정 일정이 끝나는 시간에 맞춰, 미완료 공부 계획이 있으면 알림 예약.

### 구현

```javascript
// useAppState.js에 추가
const schedulePlannerReminders = useCallback(async () => {
  if (!weeklySchedule?.enabled) return;
  if (!settingsRef.current.notifEnabled) return;
  
  const today = getTodaySchedule();
  if (!today || !today.fixed.length || !today.plans.length) return;
  
  const now = new Date();
  
  for (const f of today.fixed) {
    const [h, m] = f.end.split(':').map(Number);
    const reminderTime = new Date();
    reminderTime.setHours(h, m + 10, 0, 0); // 고정 일정 종료 + 10분
    
    if (reminderTime > now) {
      const secondsUntil = Math.floor((reminderTime - now) / 1000);
      if (secondsUntil > 0 && secondsUntil < 86400) {
        try {
          const trigger = Platform.OS === 'android'
            ? { type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL, seconds: secondsUntil, channelId: 'timer-complete' }
            : { seconds: secondsUntil };
          await Notifications.scheduleNotificationAsync({
            content: {
              title: '📚 공부 시작할 시간!',
              body: `${f.label} 끝! 오늘의 계획을 시작해볼까요?`,
              sound: true,
              ...(Platform.OS === 'android' && { channelId: 'timer-complete' }),
            },
            trigger,
          });
        } catch {}
      }
    }
  }
}, [weeklySchedule, getTodaySchedule]);

// 앱 시작 시 + 매일 자정에 호출
// 기존 초기화 useEffect 안에서 호출
useEffect(() => {
  if (!loading) schedulePlannerReminders();
}, [loading, weeklySchedule]);
```

## 11. 보완 규칙

### 이월 정책
- 미완료 계획은 **매일 리셋**. 다음 날은 그 요일의 plans가 새로 표시됨.
- 이월 기능 없음. 복잡도만 높이고 학생에게 부담감을 줌.
- "어제 미완료" 표시도 하지 않음. 매일 새 시작.

### 완료 기준
- targetMin의 **80% 이상** 공부하면 완료 체크.
- 반드시 **실제시간/목표시간** 함께 표시: "32분/40분 ✅"
- 학생이 "왜 완료야?" 혼란 없도록.

### 진행 중 표시
- 미완료이지만 공부를 시작한 항목: "12분/30분" + 작은 진행률 바
- 이러면 "조금이라도 했다"는 성취감을 줌

### 계획 초과 경고
- ScheduleEditorScreen에서 plans 합계가 가용 시간 초과 시:
  `⚠️ 계획 3h30m > 가용 2h30m`
- 부드러운 톤, 등록 자체를 막지 않음

### schoolLevel 마이그레이션
- 기존 `elementary` → `elementary_upper`로 자동 변환
- useAppState.js의 초기 로드에서 처리:
```javascript
if (s.schoolLevel === 'elementary') {
  s.schoolLevel = 'elementary_upper';
}
```

## 14. 주의사항 (절대 규칙)

- **getTheme() 시스템 반드시 유지.** 모든 색상, 폰트 크기를 theme에서 가져와.
- **기존 기능 절대 깨지면 안 됨.** 플래너를 안 쓰는 사용자는 기존과 100% 동일.
- **weeklySchedule이 null이면 플래너 미사용.** 모든 곳에서 null 체크 필수.
- **planId는 optional.** 플래너 없이 만든 타이머는 planId: null. 기존 로직 영향 없음.
- **한국어로 모든 UI 텍스트.**
- **수정할 파일만 개별 제공.** 전체 파일을 처음부터 다시 쓰지 말고, 기존 파일에 추가/수정.
- **accentColor, darkMode 등 테마 설정이 플래너 화면에도 동일하게 적용.**
